Altium Designer — Step-by-Step Process

1
Create a New PCB Project
File setup · project structure
▼
  • Open Altium → File → New → Project → PCB Project
  • Name it ClapSwitch.PrjPcb → save to a folder
  • Right-click project → Add New to Project → Schematic → save as ClapSwitch.SchDoc
  • Right-click project → Add New to Project → PCB → save as ClapSwitch.PcbDoc
You will now see 3 files in Projects panel: .PrjPcb, .SchDoc, .PcbDoc
2
Place Components on Schematic
Library search · designators
▼
  • Open .SchDoc file. Press P → Part (or use Place → Part)
  • Search Res2 in Miscellaneous Devices → place R1, R2, R3, R4 (set values in Properties)
  • Search Cap Pol2 → place C1 (10µF)
  • Search BC547 or NPN → place Q1
  • Search NE555 or LM555CN → place U1
  • Search LED → place D1. Search Battery → place B1
  • For MIC, use a 2-pin connector (CON2) or search Microphone
  • Assign designators (R1, R2...) and values in component Properties panel
Shortcut: Press Ctrl+L to open Library Search panel instantly
3
Wire the Schematic
Connections · power ports · net labels
▼
  • Press P → W (Place Wire) to draw connections between pins
  • Add VCC power port: Place → Power Port → type VCC, place on all +9V nodes
  • Add GND power port: same way, type GND, place on all ground pins
  • Use Net Labels (Place → Net Label or P → N) for long wires — e.g. label TRIG at Q1 collector and U1 pin2
  • Connect R4(1)→VCC, R4(2)→M1(+)→Q1-Base
  • Connect Q1-Collector → R3 → NE555 Pin2 and Pin4
  • Connect U1-Pin8 → VCC; U1-Pin1 → GND
  • Connect U1-Pin6 to U1-Pin2 (bistable: threshold = trigger)
  • Connect U1-Pin3 (output) → C1(+) and → R2 → D1-Anode → GND
  • Connect R1 between VCC and U1-Pin7
⚠ NE555 Pin5 (Control) → connect a 0.01µF (10nF) cap to GND. Even if not in original schematic, it's good practice to filter noise.
4
Run ERC (Electrical Rules Check)
Find wiring errors before PCB
▼
  • Go to Project → Compile PCB Project (or press Ctrl+Shift+C)
  • Then go to Design → Electrical Rules Check
  • Check the Messages panel — fix any errors (unconnected pins, missing VCC/GND)
  • Common error: "Pin not connected" on U1-Pin5 → add the 10nF cap or a no-connect marker
  • No-Connect marker: Place → No ERC Marker (use P → E) on unused pins
Warning (orange) is OK to ignore sometimes. Error (red) must be fixed before moving to PCB.
5
Transfer Schematic to PCB
Import netlist · sync components
▼
  • In schematic, go to Design → Update PCB Document (ClapSwitch.PcbDoc)
  • The Engineering Change Order (ECO) dialog opens — click Validate Changes first
  • All rows must show green tick ✓ — fix errors if any
  • Click Execute Changes → components appear in PCB editor
  • Switch to .PcbDoc → you'll see all 10 components in a cluster outside the board outline
If components don't appear, check that .SchDoc and .PcbDoc are both in the same .PrjPcb project
6
Set Board Outline & Arrange Components
Board shape · placement strategy
▼
  • Switch to Mechanical 1 or Board Shape layer → draw board outline with Place → Line (suggest: 60mm × 40mm for this circuit)
  • Or: Design → Board Shape → Define from Selected Objects
  • Drag components inside the board: B1 (battery/connector) at edge, MIC at edge (for accessibility)
  • Place U1 (NE555) in the center of the board
  • Place Q1 near MIC/R4 side, LED D1 near board edge
  • Place R1, R2, R3, R4 near their connected ICs — reduce trace length
  • Place C1 near U1 output pin (pin3)
  • Turn on View → Ratsnest (thin lines show connections) — arrange to minimize crossing ratsnest lines
Rule of thumb: Input side (MIC, Q1) on left. NE555 in center. Output (LED, C1) on right.
7
Route Traces (Copper Tracks)
Manual routing · track widths
▼
  • Press U → A (Interactive Router) or Place → Interactive Routing
  • Set track widths: Power traces (VCC, GND) = 0.5mm, Signal traces = 0.25mm
  • Route on Top Layer (red) for most signals. Use Bottom Layer (blue) if needed to avoid crossing
  • Route GND first as a polygon pour: Place → Polygon Pour on Bottom Layer → connect to GND net → flood fills entire board with copper GND plane
  • After polygon pour, press T → G → R (Repour All) to update
  • Double-click track mid-route to add vias when switching layers
  • Avoid 90° bends — use 45° angles (Altium does this automatically in interactive mode)
⚠ Keep MIC signal trace (MIC_OUT) as short as possible and away from noisy power traces to reduce false triggers.
8
DRC Check & Gerber Export
Final check · production files
▼
  • Run Tools → Design Rule Check (DRC) — must have 0 errors
  • Common DRC errors: clearance violation (move traces apart), unrouted net (finish routing), short circuit (delete overlapping track)
  • To export Gerbers: File → Fabrication Outputs → Gerber Files
  • Select layers: Top Copper, Bottom Copper, Top Silkscreen, Top Soldermask, Bottom Soldermask, Board Outline (Edge Cuts)
  • Also export: File → Fabrication Outputs → NC Drill Files (for drill holes)
  • Zip all Gerber files and send to PCB manufacturer (JLCPCB, PCBWay, etc.)
✅ Final checklist before Gerber: All pads have correct footprint → All nets fully routed → Silkscreen labels visible → Board outline closed on Mechanical/Edge layer
